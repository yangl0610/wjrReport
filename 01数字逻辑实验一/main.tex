% ==================== 实验目的 ====================
\section*{实验目的}
\addcontentsline{toc}{section}{实验目的}

本次实验为设计型实验，旨在设计与实现一个 8421BCD 码至七段数码管显示译码电路。

\begin{enumerate}
\item 理论目标：验证七段数码管译码器的真值表与逻辑表达式，掌握组合逻辑电路的分析与设计方法。
\item 实践目标：掌握 Quartus II 软件的原理图输入设计方法，学会使用波形仿真验证逻辑功能，并完成 FPGA 芯片的管脚约束与下载验证。
\end{enumerate}

% ==================== 实验环境 ====================
\section*{实验环境}
\addcontentsline{toc}{section}{实验环境}

\begin{itemize}
\item \textbf{硬件平台}：EP1C3 FPGA 开发板
\item \textbf{设计软件}：Quartus II 13.1（原理图输入法）
\item \textbf{编程接口}：USB Blaster（JTAG 模式）
\item \textbf{器件型号}：EP1C3T144C8
\end{itemize}

\subsection{1. 组合逻辑设计}

\subsubsection{1.1 设计任务}

用实验板上 4 位拨码开关作为 8421BCD 码输入（D 为最高位，A 为最低位），设计七段数码管显示驱动电路，使数码管显示对应的十进制数字（0$\sim$9）。拨码开关与 FPGA 引脚的对应关系见管脚分配表。

\subsubsection{1.2 原理分析}

七段数码管为共阳极结构，各段点亮条件为对应的阴极输入低电平（逻辑 0）。各显示数字对应的段亮灭情况如下：

注：输入 DCBA = 1010$\sim$1111 为无效 BCD 码，本设计使各段输出均为 1（共阳极下为全部熄灭）。

\subsubsection{1.3 各段输出逻辑表达式}

对有效输入（DCBA = 0000$\sim$1001），利用卡诺图化简得各段输出表达式如下（以下均为原变量表示，0 对应段亮，1 对应段灭）：

\begin{align*}
a &= D + C + BA + B'A' \\
b &= B \\
c &= \overline{C} \\
d &= D + C'B' + CBA \\
e &= D'C'A + D'B'A' \\
f &= D + CB' + CA' \\
g &= D + C + B + A
\end{align*}

\subsection{2. Quartus II 原理图设计}

\subsubsection{2.1 新建工程}

打开 Quartus II 13.1，选择 New Project Wizard，设置工程名称为 \texttt{bcd\_to\_7seg}，选择器件 EP1C3T144C8，完成工程创建。

\subsubsection{2.2 绘制原理图}

新建 Block Diagram/Schematic File，根据上述逻辑表达式从 Primitives 库中调用相应逻辑门（and、or、not）进行搭接。主要步骤如下：

\begin{enumerate}
\item 从 \texttt{primitives/logic/and} 调用多输入与门；
\item 从 \texttt{primitives/logic/or} 调用多输入或门；
\item 从 \texttt{primitives/logic/not} 调用非门；
\item 从 \texttt{primitives/pin/input} 添加 4 个输入引脚（D、C、B、A）；
\item 从 \texttt{primitives/pin/output} 添加 7 个输出引脚（a、b、c、d、e、f、g）；
\item 按逻辑表达式连接各门电路；
\item 双击输入/输出引脚，在弹出的对话框中命名管脚。
\end{enumerate}

% ==================== 实验结果分析 ====================
\section*{实验结果分析}
\addcontentsline{toc}{section}{实验结果分析}

实验结果表明，所设计的七段译码电路能够正确完成 8421BCD 码至十进制数字的显示转换。下载至 FPGA 开发板后，各数字（0$\sim$9）显示均符合真值表预期。

若出现显示异常，可能原因及修正方法如下：

\begin{itemize}
\item \textbf{数码管无显示}：检查 VCC 与 GND 是否接好，确认引脚约束与管脚对应关系是否正确；
\item \textbf{数字显示错误}：检查对应段的逻辑表达式是否写反，重新核对卡诺图化简结果；
\item \textbf{部分数字闪烁}：可能是时钟或复位信号异常，检查 JTAG 连接或重新下载。
\end{itemize}

% ==================== 实验方案的缺陷及改进意见 ====================
\section*{实验方案的缺陷及改进意见}
\addcontentsline{toc}{section}{实验方案的缺陷及改进意见}

\textbf{缺陷：}本设计仅针对有效 BCD 码（0$\sim$9）进行了处理，对于无效输入（10$\sim$15）仅简单熄灭显示，未能给出错误提示。此外，采用纯原理图设计方式，当输入变量增多时代价较高。

\textbf{改进意见：}
\begin{enumerate}
\item 引入额外的输出信号用于指示无效输入（如小数点闪烁），增强用户体验；
\item 可进一步改用 Verilog HDL 描述该译码器，减少原理图复杂度并提高可移植性；
\item 若扩展为多位数码管显示，可使用多路复用（ multiplexing）技术减少 I/O 引脚占用。
\end{enumerate}

\section*{心得体会、问题讨论}
\addcontentsline{toc}{section}{心得体会、问题讨论}

\textbf{心得体会：}通过本次实验，我初步掌握了 Quartus II 的原理图输入设计方法，体会到了组合逻辑电路从真值表到逻辑表达式再到硬件实现的全过程。在卡诺图化简过程中，我加深了对与或表达式最小化问题的理解。FPGA 下载验证环节也让我认识到理论设计与实际硬件之间可能存在的差异。

\textbf{问题讨论：}

\begin{enumerate}
\item 为什么七段数码管采用共阳极结构？在实际设计中应如何选择？
\item 若将输入改为 2421BCD 码或余 3 码，译码电路应如何修改？
\item 在多位数码管显示系统中，多路复用（扫描）方式的工作原理是什么？有何优缺点？
\end{enumerate}
